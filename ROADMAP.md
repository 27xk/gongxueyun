# Roadmap

本文档记录后续计划。优先级会根据实际使用反馈调整。

## 近期计划

- **补卡体验完善：** 展示每次补卡请求的明细结果，包括日期、类型、状态和失败原因。
- **部署体验完善：** 提供包含 MySQL 的完整 Docker Compose 示例，降低首次部署门槛。
- **配置导入导出：** 支持导出用户配置模板，并批量导入用户。
- **执行日志增强：** 支持按任务类型、执行状态、用户备注过滤。
- **文档截图维护：** 每个核心页面保留最新截图，并在版本发布时同步更新。

## 中期计划

- **通知渠道扩展：** 在 SMTP 之外支持更多推送渠道。
- **报告模板管理：** 支持自定义日报、周报、月报 Prompt 和模板。
- **批量任务看板：** 提供更细粒度的批量任务进度、失败重试和导出能力。
- **权限细化：** 将管理端角色权限拆得更细，便于多人协作。
- **测试体系扩展：** 增加前端组件测试和接口级集成测试。

## 长期计划

- **多租户隔离：** 支持更清晰的团队 / 租户边界。
- **多平台适配：** 在现有工学云能力之外，抽象更多实习平台接入层。
- **可观测性：** 增加指标、健康检查和结构化日志。
- **镜像发布稳定化：** 发布稳定版本镜像和升级说明。

## 不在近期范围

- 不内置公开在线 Demo 账号。
- 不提供对第三方平台规则的规避说明。
- 不把前端测试、端到端测试和可观测性一次性全部补齐。

## 反馈方式

- Bug：使用 GitHub Issue 的 Bug 模板。
- 功能建议：使用 Feature Request 模板。
- PR：请先阅读 [CONTRIBUTING.md](./CONTRIBUTING.md)。
